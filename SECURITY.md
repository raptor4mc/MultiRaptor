# Security policy

## Supported versions

Security fixes are provided for:

- Latest stable minor line (current)
- One previous LTS line (when designated)

See `docs/lts_support_policy.md` for lifecycle timing.

## Reporting a vulnerability (responsible disclosure)

Please do **not** open public issues for undisclosed vulnerabilities.

- Contact: `security@magphos.dev`
- Include: affected version, reproduction steps, impact, and suggested mitigation.
- Optional: encrypted report with your public key fingerprint in the email body.

## Response SLA

- Initial acknowledgement: **within 2 business days**
- Triage decision: **within 5 business days**
- Coordinated fix timeline shared after triage.

## CVE process

- Valid vulnerabilities are tracked with internal advisory IDs and (when applicable) CVE IDs.
- Security releases include:
  - affected versions,
  - severity,
  - mitigation/workaround,
  - patched versions.
