# Design

Language evolution workspace for MagPhos.

Use this folder to track the intent and status of language/runtime changes before implementation.

## Structure
- `proposals/` – active design proposals.
- `rejected/` – rejected proposals with rationale.
- `future-syntax/` – syntax sketches and parsing implications.
- `bytecode-evolution/` – VM/IR/bytecode planning notes.

## Suggested proposal format
1. Problem statement
2. Goals / non-goals
3. User-facing syntax or API
4. Runtime/compiler impact
5. Migration strategy
6. Alternatives considered
