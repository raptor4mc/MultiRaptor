# design/

This workspace tracks MagPhos language and runtime evolution decisions over time.

## Structure

- **proposals/**: active proposals under discussion.
- **rejected/**: proposals that were evaluated but declined.
- **future-syntax/**: candidate syntax directions and drafts.
- **bytecode-evolution/**: VM/bytecode format ideas and migration notes.

## Proposal template (recommended)

For new items in `design/proposals/`, include:

1. Problem statement
2. Proposed change
3. Alternatives considered
4. Backward-compatibility impact
5. Implementation notes
6. Decision status and date
