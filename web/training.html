1) Full Language Map (Core + Extended)

This training covers the core MagPhos building blocks: declarations, functions, control flow, modules, expressions, safety rules, and debugging workflows. Use this as your complete learning map.

Declarations & Assignment

Use var and const. Assignment updates existing names.

var hp = 100
const maxHp = 100
hp = hp - 10

Functions

Use fn name(args) { ... }. Return values with return.

fn heal(current, amount) {
  return current + amount
}

Conditionals

Branch with if/else, and when for conditional blocks.

if hp <= 0 {
  print "game over"
} else {
  print "fight"
}

Loops

Use while, for, loop, repeat while.

for (var i = 0; i < 3; i = i + 1) {
  print i
}

Switch/Match

Choose by value with switch or match and case.

switch state {
  case "idle" { print "wait" }
  default { print "run" }
}

Try/Catch

Guard risky logic with try { ... } catch { ... }.

try {
  print dangerous()
} catch {
  print "recovered"
}

Modules

Import module namespaces or use file-based module scripts.

import game.engine
use "combat_utils.mp"

Ask/Set + IO style

ask writes input into existing vars. set updates existing vars.

var name = ""
ask "Name?" -> name
set name = name + "!"

Expressions & Literals

Numbers, strings, booleans, null, array literals, arithmetic and logical operators.

var ok = (score >= 10) and not failed
var loot = ["coin", "potion", "key"]

Namespaces + visibility

Use namespace blocks and visibility tags like public/private.

public fn boot() {
  print "ready"
}

2) How your code runs

Lexer phase

Converts source text into tokens.

Parser phase

Builds structured statements and expressions.

Semantic phase

Rejects undefined names, bad returns, and duplicate declarations.

Runtime phase

Executes valid code and surfaces structured runtime errors.

Predictability contract

Same source + environment should produce deterministic behavior.

Why it matters for games

Reliable simulation behavior and easier debugging of gameplay state.
