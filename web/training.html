How MagPhos code moves from source to output

1) Write code in .mp

MagPhos programs use readable syntax with explicit blocks and control statements like if, while, for, match, and functions with fn.

fn greet(name) { print "Hi " + name } var player = "Raptor" greet(player)

2) Compiler pipeline checks your program

  1. Lexer reads characters and makes tokens.
  2. Parser builds syntax structures from tokens.
  3. Semantic analysis validates rules like declared variables and valid return usage.
  4. Runtime/interpreter executes valid code and returns output.

3) Predictability rules (important)

  • Variables must be declared before assignment.
  • return is only valid inside functions.
  • Same input and environment should produce deterministic behavior.

4) Why this helps game scripting

  • Fewer hidden side effects during gameplay logic.
  • Easier debugging for quests, AI behavior, and events.
  • Consistent behavior between runs (great for testing and replays).

5) Training checklist

  • Create a variable with var and update it safely.
  • Write one function with parameters and a return.
  • Add one loop and one conditional branch.
  • Run and confirm output in the Playground.
